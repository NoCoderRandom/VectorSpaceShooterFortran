# Prompt 12 — Tests, docs, and repo hardening

Make the repository strong.

Add:
- focused tests for math/projection where practical
- tests for model/line pipeline where practical
- dependency/setup documentation
- architecture overview
- controls
- build/run instructions
- known limitations
- next-step ideas

Also:
- clean file names
- clean comments
- no temporary junk
- no AI logs
- no personal data
- no misleading README claims

Verify:
- a fresh user can understand the layout
- the repo looks professional
- build instructions are reproducible
